# Review Procedure

## Purpose
Review current diff for correctness, scope drift, and validation readiness.

## Rules
- Review actual diff and evidence
- Classify: BLOCKER / MAJOR / MINOR / QUESTION
- Check validation evidence and ownership
- Do not make unrelated changes
- Readonly reviewer role should be used when available

## Output
```text
Review Result

BLOCKER:
MAJOR:
MINOR:
QUESTION:
Validation Status:
Recommended Fixes:
```
