# Plan Procedure

## Goal
Turn a request into a scoped, verifiable plan before non-trivial edits.

## Rules
- Inspect relevant files first
- Keep scope tight
- Include validation commands
- Update `current-task.md` and `harness/session/session-state.json`
- Do not implement inside plan

## Output
```text
Plan

Goal:
Scope:
Non-Scope:
Files Likely Affected:
Steps:
Validation:
Risks:
```
