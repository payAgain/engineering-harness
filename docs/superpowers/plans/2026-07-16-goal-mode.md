# Bounded Goal Execution Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make Human-confirmed Scope start a bounded Goal by default, allowing agents to issue and accept multiple independently committed Builds until the Goal is evidenced as complete or requires Human escalation.

**Architecture:** Add repository-owned Goal and Goal-Acceptance templates, then extend the existing Build authorization schema with a `goal-delegation` alternative to Human Build approval. Keep Goal control in protocol/template roles rather than adding a long-running Python engine; the Python CLI and project-local checker validate persisted authorization, state, evidence, and compatibility contracts so any supported coding agent can execute the loop from repository state.

**Tech Stack:** Python 3.10+ standard library (`json`, `re`, `pathlib`, `unittest`), Markdown/YAML/JSON runtime templates, `argparse` CLI, GitHub Flow.

## Global Constraints

- One Initiative may have at most one active Goal in MVP.
- Scope confirmation defaults to `execution_mode: goal`; an explicit `execution_mode: build-by-build` preserves per-Build Human approval.
- Goal-delegated Builds retain manifest, role pipeline, verification, acceptance evidence, and one verified commit SHA per Build.
- Goal execution stops on acceptance or escalation and never performs push, PR creation, merge, tag, release, protected-branch updates, production operations, credential operations, or other unauthorized outward actions.
- Goal objective, required success criteria, `in_scope`, and `out_of_scope` cannot be silently changed by the agent; substantive change requires Human confirmation and a new Scope revision or Goal.
- Goal evaluation has exactly three decisions: `continue`, `achieved`, or `escalate`.
- Runtime SSOT remains in the initialized target repository; integrations do not become runtime SSOT.
- Windows remains the primary platform; use Python 3.10+ and add no third-party dependency.
- Preserve LF-authored portable content and do not introduce machine-local absolute paths.
- Do not change `VERSION` or `pyproject.toml` unless release work is separately authorized.

- [ ] **Step 3: Write the canonical Goal reference**

Create `protocol/references/goals.md` with these normative sections and exact rules:

```markdown
# Goal Mode

## 1. Definition

A **Goal** (`G-00x`) is the bounded, revocable execution authorization created by Human Scope confirmation. One Initiative has at most one active Goal. Scope confirmation defaults to `execution_mode: goal`; explicit `execution_mode: build-by-build` retains Human approval for every Build.

Scope complete ≠ Intent satisfied. Build accepted ≠ Goal accepted.

## 2. Authorization

`human Scope confirmation → Goal G-00x → goal-delegation Build B-00x`.
A Build uses exactly one authorization type: `human-build-approval` or `goal-delegation`. Goal delegation is not a fake Human approval.

## 3. Loop

`plan/replan → containment → Build → Accept → evaluate`.
Evaluation is exactly `continue | achieved | escalate`. `continue` issues the next bounded Build; `achieved` creates Goal Acceptance; `escalate` stops before another Build.

## 4. Containment

Every delegated Phase maps to an existing required success criterion, stays in `in_scope`, avoids `out_of_scope`, stays within budgets, and requires no unapproved outward, irreversible, credential, production, security/privacy, or major contract decision.

## 5. Replan

The controller may split, merge, reorder, add, or remove unstarted Phases only to satisfy existing criteria. Every change increments `plan_revision`, records reason/evidence, and reruns containment. It never rewrites an accepted Phase or historical Build.

## 6. Escalation

Escalate on Scope change, high-risk/outward action, major product or architecture choice, contradictory/unachievable criteria, exhausted blocker budget, no-progress budget, replan budget, failed verification, or unsafe recovery. Default budgets are 3 consecutive failures per blocker, 2 no-progress Builds, and 5 replans without Human.

## 7. Completion and stop

Goal accepted requires every required criterion to be `met` with evidence and commit SHA, project checks and observed flows passing, no unresolved blocker, a safe worktree, and no unauthorized Ship action. Stop on the local working branch; push, PR, merge, tag, release, protected branches, and production remain Human gates.

## 8. Recovery

Restore branch/worktree, open Initiative, the one active Goal, `active_build_id`, invocations, evidence, and commit SHAs. Resume an active Build instead of issuing another. Contradictory state becomes `escalation_required`; never guess.
```

```python
def _yaml_scalar(text: str, key: str) -> str | None:
    match = re.search(rf"(?m)^{re.escape(key)}:\s*([^\n#]+)", text)
    return match.group(1).strip() if match else None


def _goal_criteria(text: str) -> set[str]:
    return set(re.findall(r"(?m)^\s{2}- id:\s*(SC-\d{3,})\s*$", text))


def _load_goals(root: Path, problems: list[str]) -> dict[str, dict[str, object]]:
    goals: dict[str, dict[str, object]] = {}
    active_by_initiative: dict[str, str] = {}
    for path in (root / "harness" / "goals").glob("G-*.yaml"):
        text = path.read_text(encoding="utf-8")
        goal_id = _yaml_scalar(text, "goal_id")
        initiative_id = _yaml_scalar(text, "initiative_id")
        status = _yaml_scalar(text, "status")
        revision_match = re.search(r"(?m)^scope:\s*\n\s{2}revision:\s*(\d+)\s*$", text)
        criteria = _goal_criteria(text)
        if not goal_id or not re.fullmatch(r"G-\d{3,}", goal_id) or not initiative_id or not re.fullmatch(r"I-\d{3,}", initiative_id):
            problems.append(f"INVALID GOAL: {path.relative_to(root)}: goal_id/initiative_id")
            continue
        if status not in {"draft", "awaiting_scope_confirmation", "active", "achieved", "accepted", "paused", "blocked", "escalation_required", "cancelled"}:
            problems.append(f"INVALID GOAL: {path.relative_to(root)}: status")
        if not revision_match or not criteria:
            problems.append(f"INVALID GOAL: {path.relative_to(root)}: scope revision/success criteria")
        if status == "active" and initiative_id in active_by_initiative:
            problems.append(f"MULTIPLE ACTIVE GOALS: {initiative_id}")
        elif status == "active":
            active_by_initiative[initiative_id] = goal_id
        goals[goal_id] = {
            "initiative_id": initiative_id,
            "status": status,
            "scope_revision": int(revision_match.group(1)) if revision_match else None,
            "criteria": criteria,
        }
    return goals
```

```python
def _goal_runtime_summary(target: Path) -> tuple[str | None, str | None, str | None]:
    goals = sorted((target / "harness" / "goals").glob("G-*.yaml"))
    if not goals:
        return None, None, None
    text = goals[-1].read_text(encoding="utf-8")
    goal_id = re.search(r"(?m)^goal_id:\s*(G-\d{3,})\s*$", text)
    status = re.search(r"(?m)^status:\s*([a-z_]+)\s*$", text)
    stage = re.search(r"(?m)^loop_stage:\s*([a-z_]+)\s*$", text)
    return (
        goal_id.group(1) if goal_id else None,
        status.group(1) if status else None,
        stage.group(1) if stage else None,
    )
```
